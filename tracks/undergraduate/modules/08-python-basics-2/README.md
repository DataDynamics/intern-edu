## 📚 본문

### 함수 — 일에 이름을 붙여 재사용하기

같은 코드를 여기저기 복사해 붙이다 보면, 나중에 고칠 때 모든 사본을 일일이 찾아 고쳐야 합니다. **함수**는 이 문제를 푸는 도구입니다. 한 번 정의해 두면 이름만 불러 몇 번이고 다시 쓸 수 있고, 고칠 일이 생겨도 한 곳만 고치면 됩니다. `def`로 정의하고, `return`으로 결과를 돌려줍니다.

```python
def add(a, b):
    return a + b

def greet(name, greeting="안녕"):    # 기본값을 둔 인자
    return f"{greeting}, {name}!"

add(2, 3)            # 5
greet("인턴")         # "안녕, 인턴!"
greet("인턴", "환영")  # "환영, 인턴!"
```

좋은 함수의 원칙은 **하나의 함수는 하나의 일만** 하도록 작게 쪼개는 것입니다. 함수가 짧고 이름이 분명하면, 코드를 읽는 사람은 그 안을 들여다보지 않고 이름만으로 무슨 일을 하는지 짐작할 수 있습니다. 참고로 `return`이 없는 함수는 아무것도(`None`을) 돌려주지 않으며, 인자에 기본값을 주면 호출할 때 그 값을 생략할 수 있습니다.

### 리스트 — 순서가 있는 묶음

리스트는 여러 값을 **순서대로** 담는 그릇입니다. 대괄호로 만들고, 0부터 시작하는 번호(인덱스)로 원소에 접근합니다. 뒤에서부터 셀 때는 음수 인덱스를 쓰고, 일부 구간을 잘라낼 때는 슬라이싱을 씁니다.

```python
nums = [10, 20, 30]
nums.append(40)     # 끝에 추가 → [10, 20, 30, 40]
nums[0]             # 10  (첫 번째)
nums[-1]            # 40  (마지막)
nums[1:3]           # [20, 30]  (1번부터 3번 직전까지)
```

리스트를 다룰 때 알아 두면 코드가 훨씬 간결해지는 표현이 **리스트 컴프리헨션**입니다. "기존 리스트의 각 원소를 변형하거나 걸러서 새 리스트를 만든다"는 작업을 한 줄로 압축해 줍니다.

```python
squares = [n * n for n in range(5)]          # [0, 1, 4, 9, 16]
evens = [n for n in nums if n % 20 == 0]     # 조건에 맞는 것만 추리기
```

### 딕셔너리 — 이름표가 달린 저장소

데이터 엔지니어링에서 가장 자주 쓰는 자료구조가 바로 딕셔너리입니다. 리스트가 번호로 값을 찾는다면, 딕셔너리는 **키(key)라는 이름표**로 값을 찾습니다. 한 건의 레코드(예: 사용자 한 명의 정보)를 표현하거나, 무언가의 개수·합계를 집계할 때 빛을 발합니다.

```python
user = {"name": "intern", "age": 22}
user["name"]               # "intern"
user["city"] = "Seoul"     # 새 키 추가
user.get("phone", "없음")   # 키가 없으면 기본값 반환 → 에러 방지
```

여기서 `user["phone"]`처럼 없는 키를 대괄호로 직접 꺼내려 하면 프로그램이 오류로 멈춥니다. 그래서 키가 있을지 확신할 수 없을 때는 `.get(키, 기본값)`을 쓰는 것이 안전합니다. 이 `.get` 패턴은 **집계**에서 특히 강력합니다. 아래는 단어들이 몇 번씩 나오는지 세는 전형적인 코드로, 데이터 엔지니어가 평생 변형해 가며 쓰게 될 패턴입니다.

```python
words = ["a", "b", "a", "c", "a"]
counts = {}
for w in words:
    counts[w] = counts.get(w, 0) + 1     # 없으면 0에서 시작해 1씩 더함
# counts == {'a': 3, 'b': 1, 'c': 1}
```

### set과 tuple — 나머지 두 그릇

자주 쓰는 자료구조 둘을 더 알아 둡시다. **set(집합)**은 중복을 허용하지 않아, 무언가를 유일하게 추리거나 "이미 본 것"을 기억할 때 유용합니다. **tuple(튜플)**은 한번 만들면 바꿀 수 없는 묶음으로, 좌표 `(위도, 경도)`처럼 함께 다녀야 하고 변하면 안 되는 값에 적합합니다.

| 자료구조 | 순서 | 변경 | 중복 | 대표 용도 |
|----------|:----:|:----:|:----:|-----------|
| list  | O | O | O | 순서 있는 데이터 |
| dict  | O | O | 키는 unique | 매핑·집계·레코드 |
| set   | X | O | X | 중복 제거·집합 연산 |
| tuple | O | X | O | 변하면 안 되는 묶음 |

상황에 맞는 그릇을 고르는 안목이 곧 실력입니다. "순서가 중요한가, 중복을 막아야 하나, 이름표로 찾을 건가"를 떠올리면 자연스럽게 답이 나옵니다.

### 예외 처리 — 더러운 데이터에 대비하기

교과서의 데이터는 깔끔하지만, 현실의 데이터는 그렇지 않습니다. 빈칸, 깨진 글자, 숫자여야 할 자리에 들어온 `"abc"`가 섞여 있지요. 이런 값 하나 때문에 수만 건을 처리하던 프로그램이 통째로 멈춘다면 곤란합니다. **예외 처리**는 "위험할 수 있는 코드를 `try`로 감싸고, 문제가 생기면 `except`로 받아 처리한다"는 안전장치입니다.

```python
def to_int(value):
    try:
        return int(value)
    except ValueError:
        return None        # 숫자로 못 바꾸면 None을 돌려주고 멈추지 않는다

to_int("42")    # 42
to_int("abc")   # None
```

이 사고방식은 앞 모듈의 "프로그램이 죽지 않게 한다"는 정신의 연장입니다. 데이터 파이프라인에서는 한 건의 불량 데이터가 전체를 멈추게 두기보다, 안전하게 건너뛰고 따로 기록해 두는 방식이 훨씬 중요합니다.

### 모듈 — 코드를 나누고 가져다 쓰기

코드가 길어지면 한 파일에 다 욱여넣기보다 여러 파일로 나누는 편이 낫습니다. 이렇게 나눈 각 파일을 **모듈**이라 하고, `import`로 다른 파일의 함수를 가져다 씁니다. 또 Python에는 표준 라이브러리라는 강력한 도구 상자가 기본 제공되어, 굳이 직접 만들지 않아도 되는 기능이 많습니다.

```python
from utils import to_int            # 내가 만든 utils.py의 함수 가져오기
import json, csv                    # 표준 라이브러리
from collections import Counter

Counter(["a", "b", "a"])            # Counter({'a': 2, 'b': 1}) — 집계를 한 줄로
```

앞서 손으로 짠 `.get` 집계 패턴이 `Counter` 한 줄로 끝나는 것을 보면, "이미 누군가 잘 만들어 둔 도구를 찾아 쓰는 것"도 중요한 실력임을 알 수 있습니다.
