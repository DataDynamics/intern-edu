# 모듈 00 — AI 시대, 왜 취업이 어려운가

> **포커스**: AI가 바꾼 신입 개발자 시장의 현실 진단
> **예상 기간**: 2~3일 (오리엔테이션)
> **형식**: 강의 + 자기진단 + 조별 토론 (코드 없음)

이 모듈의 목적은 **겁주기가 아니라 방향 잡기**입니다. 막연한 불안을 구체적인 현실 인식으로
바꾸고, "그래서 나는 무엇을 해야 하는가"라는 질문으로 연결합니다. 이 질문에 대한 답이
앞으로의 13주를 이끄는 동력이 됩니다.

---

## 🎯 학습 목표
- AI가 신입 개발자 채용 시장을 어떻게 바꿨는지 구체적으로 이해한다
- "대체되기 쉬운 일"과 "가치가 오르는 일"을 구분한다
- 데이터 직군이 상대적으로 견고한 이유를 설명할 수 있다
- 자신의 현재 상태를 객관적으로 진단한다

---

## 📚 핵심 내용

### 1. 생성형 AI가 바꾼 신입 개발자 시장
- 단순 코딩·보일러플레이트·CRUD는 AI가 빠르게 생성 → "코드를 칠 줄 안다"는 것의 희소가치 하락
- 기업은 신입에게도 **더 높은 기준**을 요구 (이른바 "경력 같은 신입")
- 주니어가 맡던 반복 업무 일부를 AI가 흡수하면서 **주니어 채용 문이 좁아짐**

### 2. 양극화 — 도구로 쓰는 사람 vs 대체되는 사람
- AI를 **레버리지**로 써서 생산성을 몇 배 높이는 사람
- AI가 하는 일밖에 못 해서 **대체되는** 사람
- 같은 신입이라도 이 격차가 점점 벌어진다

### 3. 무엇의 가치가 떨어지고, 무엇이 오르는가
| 가치 하락 ↓ | 가치 상승 ↑ |
|-------------|-------------|
| 문법 암기, 단순 구현 | 문제 정의 (무엇을 풀지 정하기) |
| "검색해서 붙여넣기" | 데이터·도메인 이해 |
| 혼자 빨리 치는 코딩 | 시스템 설계·디버깅·협업 |
> 핵심: **"코딩 자체"의 가치는 떨어져도 "문제 해결 능력"의 가치는 오른다.**

### 4. 왜 데이터 직군인가 (이 교육의 방향성)
- 데이터는 **회사마다 고유**하다 — 일반 AI가 그 맥락을 알지 못한다
- 데이터 품질·파이프라인 신뢰성·도메인 해석은 자동으로 대체하기 어렵다
- 그래서 데이터 엔지니어링은 AI 시대에 상대적으로 **견고한 커리어**다
- 이 교육은 바로 그 견고한 축을 만드는 과정이다
