# 모듈 02 — 데이터 엔지니어란?

> **포커스**: DE 직무 소개, 데이터 흐름의 큰 그림, 학습 로드맵
> **예상 기간**: 2~3일 (오리엔테이션)
> **형식**: 강의 + 데이터 흐름 지도 그리기 (코드 없음)
> **선행 모듈**: 00~01

본격적인 기술 학습에 들어가기 전, **데이터 엔지니어가 실제로 무슨 일을 하는지**, 그리고
앞으로 배울 것들이 전체 그림에서 어디에 위치하는지 지도를 그립니다.

---

## 🎯 학습 목표
- 데이터 엔지니어의 역할과 다른 데이터 직군과의 차이를 안다
- 데이터가 흐르는 큰 그림(수집→저장→처리→활용)을 이해한다
- 이 교육의 각 모듈이 그 그림 어디에 해당하는지 연결한다

---

## 📚 핵심 내용

### 1. 데이터 직군 비교
| 직군 | 한 줄 역할 |
|------|-----------|
| **데이터 엔지니어 (DE)** | 데이터를 **모으고·옮기고·다듬는 파이프라인**을 만든다 |
| 데이터 분석가 (DA) | 데이터로 **질문에 답하고 인사이트**를 낸다 |
| 데이터 과학자 (DS) | 데이터로 **모델·예측**을 만든다 |
| ML 엔지니어 | 모델을 **서비스로 배포·운영**한다 |
> DE는 다른 모든 직군이 **신뢰할 수 있는 데이터를 쓰도록** 토대를 만드는 사람.

### 2. 데이터가 흐르는 큰 그림
```
[수집]        [저장]            [처리/변환]        [활용]
소스 ──────▶ 데이터 저장소 ──────▶ 파이프라인 ──────▶ 분석/대시보드/ML
(API,로그,DB)  (DB, 웨어하우스)    (ETL/ELT)         (BI, 모델)
```
- **수집(Extract)**: API·로그·DB·파일에서 데이터를 가져온다
- **저장(Load)**: 데이터베이스/웨어하우스에 적재한다
- **변환(Transform)**: 정제·집계·가공해 쓸 수 있는 형태로 만든다
- **활용**: 분석가·과학자·서비스가 그 데이터를 사용한다

### 3. DE가 매일 쓰는 도구 (= 이 교육의 내용)
| 영역 | 도구 | 이 교육 모듈 |
|------|------|--------------|
| 운영체제·환경 | 터미널, Linux, Docker | 03, 05, 12 |
| 문서화 | Markdown | 04 |
| 협업 | Git/GitHub | 06 |
| 프로그래밍 | Python | 07, 08, 09 |
| 데이터 질의 | SQL | 10, 11 |
| 데이터 가공 | pandas | 13 |
| 분석·전달 | EDA, 시각화 | 14 |
| 종합 | 미니 프로젝트 | 15 |
> 즉, 이 교육의 모든 모듈은 "데이터 흐름을 만드는 사람"이 되기 위한 조각들이다.

### 4. 신입 데이터 엔지니어의 하루 (예시)
- 어젯밤 파이프라인이 잘 돌았는지 로그 확인 (Linux, 모듈 05)
- 데이터가 이상하면 SQL로 원인 추적 (모듈 10~11)
- 새 데이터 소스를 붙이는 코드 작성 + PR (Python·Git, 모듈 06~09)
- 분석가 요청에 맞춰 데이터 정제 (pandas, 모듈 13)
