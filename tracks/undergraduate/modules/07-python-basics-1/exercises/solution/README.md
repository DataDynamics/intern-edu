# 정답 해설 — 상태코드 통계기

## classify
범위 비교는 `200 <= status < 300` 처럼 **연쇄 비교**로 깔끔하게 씁니다.
`elif`로 위에서부터 걸러지므로 순서가 중요합니다.

## summarize
1. 카테고리별 0으로 초기화된 딕셔너리를 만든다.
2. 리스트를 `for`로 순회하며 각 상태코드를 `classify`로 분류한다.
3. 분류 결과를 키로 `counts[category] += 1`.

`classify`를 재사용한 덕분에 `summarize`가 짧고 명확해집니다. (함수 분리의 가치 →
모듈 08에서 더 다룹니다.)
