# 정답 예시 — 첫 Pull Request

핵심은 **결과물**보다 **흐름**입니다. 아래가 모두 맞으면 통과입니다.

1. `git switch -c feature/...` 로 브랜치에서 작업했는가
2. 변경이 자기소개 파일 하나로 깔끔한가 (불필요한 파일 X)
3. 커밋 메시지가 명령형이고 의미가 분명한가
4. PR 제목/설명에 "무엇을/왜"가 담겼는가

## PR 설명 좋은 예
```
제목: Add intro for gildong

본문:
- 05-git-collaboration 실습용 자기소개 파일 추가
- 데이터 엔지니어 교육 첫 PR
```

`students/gildong.md`가 작성 예시입니다.
