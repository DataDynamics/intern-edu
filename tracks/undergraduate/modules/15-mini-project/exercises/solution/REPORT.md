# 판매 데이터 분석 리포트 (작성 예시)

## 1. 데이터 품질 메모
- **카테고리 대소문자 불일치** (Book/BOOK/book) → 모두 소문자로 통일
- **앞뒤 공백** (' 데이터중심설계 ', 'toy ', ' cash') → strip 처리
- **금액 결측/형식오류** (라면 빈 값, 블록 'abc') → 숫자 변환 실패 2건 제거
- **완전 중복 행** (리팩토링 22000 card 1건) → 중복 제거
- 결과: 원본 14행 → 정제 후 **11행**

## 2. 분석 질문과 답
1. 매출이 가장 높은 카테고리는? → **book (67,000)**, 이어 toy(50,000), food(17,500)
2. 날짜별 매출 추이는? → 06-02(41,500)가 가장 높고 06-03(26,500)이 가장 낮음
3. 결제수단 비중은? → **card 7건 / cash 4건** (카드가 약 64%)

## 3. 발견 사실 3가지
1. book이 전체 매출의 절반(67,000 / 134,500 ≈ 50%)을 차지한다.
2. food는 건수(4건)는 많지만 객단가가 낮아 매출 기여가 가장 작다.
3. 원본의 약 21%(3/14행)가 품질 문제로 제거·정제되었다 — 데이터 검증의 중요성.

## 4. 한계와 다음 단계
- 표본이 작아 요일 패턴을 일반화하기 어렵다.
- 공공데이터포털의 실제 판매 데이터로 확장하면 추세 분석이 가능하다.
