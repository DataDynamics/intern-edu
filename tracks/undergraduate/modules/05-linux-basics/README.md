> **포커스**: 파일시스템·권한·핵심 명령어·파이프/리다이렉션·쉘 스크립트
> **예상 기간**: 1주
> **선행 모듈**: 03 개발 환경 & 터미널

데이터 엔지니어가 매일 발 딛고 서 있는 땅은 대부분 Linux입니다. 데이터가 흐르는 서버, 파이프라인이 도는 실행 환경, 클라우드 인스턴스, 그리고 우리가 곧 배울 Docker 컨테이너까지 — 거의 모두 Linux 위에서 돌아갑니다. 그래서 마우스 없이 **터미널만으로** 파일을 다루고 데이터를 가공하고 반복 작업을 자동화하는 능력은, 데이터 엔지니어에게 선택이 아니라 기본기입니다.

다행히 처음부터 모든 것을 외울 필요는 없습니다. 이 모듈에서는 길을 찾아 이동하고, 파일을 만들고 옮기고, 텍스트를 가공하고, 명령들을 이어 붙이고, 권한을 다루고, 짧은 자동화 스크립트를 쓰는 — 가장 많이 쓰는 핵심만 손에 익힙니다. Windows를 쓴다면 **WSL2 + Ubuntu**, macOS라면 기본 터미널에서 그대로 따라오면 됩니다.

## 📚 본문

### 파일시스템 — 하나의 큰 나무

Linux의 모든 파일은 최상위 루트(`/`)에서 뻗어 나온 하나의 거대한 나무에 매달려 있습니다. 내 작업 공간인 홈 디렉토리는 `~`로 줄여 부르고, 시스템 설정은 `/etc`, 로그는 `/var/log`, 임시 파일은 `/tmp`에 담깁니다. 특히 `/var/log`는 데이터 엔지니어가 "어젯밤 파이프라인이 잘 돌았나"를 확인하러 자주 들르는 곳입니다.

위치를 가리키는 방식에는 두 가지가 있습니다. 루트부터 전체 길을 적는 **절대경로**(`/home/intern/data.csv`)와, 지금 서 있는 자리를 기준으로 적는 **상대경로**(`./data.csv`, `../other/`)입니다. 여기서 `.`은 현재 디렉토리, `..`은 한 단계 위, `~`은 홈을 뜻합니다. 이 약속만 알면 길 찾기는 어렵지 않습니다.

```bash
pwd              # 지금 어디에 있는지
ls -al           # 숨김 파일까지, 권한·크기·날짜를 자세히
cd /var/log      # 그 디렉토리로 이동
cd ..            # 한 단계 위로
cd ~             # 홈으로
```

### 파일과 디렉토리 다루기

파일을 만들고, 복사하고, 옮기고, 지우는 일은 가장 기본적인 작업입니다. 디렉토리를 만들 때 중간 경로까지 한 번에 만들려면 `-p`를, 디렉토리를 통째로 복사하거나 지우려면 `-r`을 붙입니다.

```bash
mkdir -p data/raw     # 중간 경로까지 한 번에
touch report.txt      # 빈 파일 만들기
cp a.csv b.csv        # 복사
mv old.txt new.txt    # 이름 변경 또는 이동
rm report.txt         # 삭제
```

여기서 꼭 명심할 것이 있습니다. **Linux의 `rm`에는 휴지통이 없습니다.** 지우면 곧바로 사라집니다. 특히 `rm -rf` 같은 강력한 명령은 시스템을 통째로 망가뜨릴 수 있으니, 경로를 두 번 확인하는 습관을 들이세요.

### 파일 내용 들여다보기

파일을 열어 보는 방법도 여러 가지입니다. 전체를 한 번에 쏟아내는 `cat`, 페이지 단위로 넘겨 보는 `less`, 앞·뒤 몇 줄만 보는 `head`·`tail`이 있습니다. 그중 데이터 엔지니어가 특히 사랑하는 명령이 `tail -f`인데, 실시간으로 쌓이는 로그를 그 자리에서 따라가며 볼 수 있어 장애 대응 때 큰 힘이 됩니다.

```bash
head -n 5 access.log   # 앞 5줄
tail -f app.log        # 실시간으로 추가되는 로그 따라가기
wc -l access.log       # 줄 수 세기
```

### 검색 — grep과 find

수많은 줄 속에서 원하는 것을 찾을 때는 `grep`이, 수많은 파일 속에서 특정 파일을 찾을 때는 `find`가 나섭니다. `grep`은 패턴이 들어간 줄을 골라 주고, 옵션으로 대소문자 무시(`-i`)나 개수 세기(`-c`), 폴더 전체 재귀 검색(`-r`)을 더할 수 있습니다.

```bash
grep "ERROR" app.log          # ERROR가 든 줄 찾기
grep -c "ERROR" app.log       # 그런 줄이 몇 개인지
find . -name "*.csv"          # 현재 폴더 이하의 모든 csv 파일
```

### 파이프와 리다이렉션 — 데이터 엔지니어의 사고방식

이 모듈에서 가장 중요한 개념입니다. Linux의 철학은 "작은 도구 하나가 한 가지 일을 잘하게 만들고, 그것들을 이어 붙여 큰일을 한다"는 것입니다. 그 이어 붙이는 장치가 **파이프(`|`)**로, 앞 명령의 출력을 뒤 명령의 입력으로 흘려보냅니다. 그리고 결과를 화면이 아니라 파일로 보내고 싶을 때는 **리다이렉션**(`>` 덮어쓰기, `>>` 이어붙이기)을 씁니다.

아래 한 줄은 접속 로그에서 상태코드만 뽑아(`awk`), 정렬한 뒤(`sort`), 같은 것끼리 세고(`uniq -c`), 많은 순으로 다시 정렬해(`sort -rn`), 상위 5개만 보여 줍니다(`head`). 작은 도구들이 데이터를 한 방향으로 흘려보내며 가공하는 이 흐름이, 바로 데이터 파이프라인의 축소판입니다.

```bash
awk '{print $9}' access.log | sort | uniq -c | sort -rn | head -5
```

여기서 한 가지 요령을 기억하세요. `uniq`는 **인접한** 중복만 묶기 때문에, 항상 `sort`로 같은 값을 모은 다음에 써야 합니다. `sort | uniq -c`는 한 단어처럼 붙어 다니는 짝입니다.

### 권한 — 누가 무엇을 할 수 있는가

`ls -l`을 치면 줄 맨 앞에 `-rwxr-xr--` 같은 기호가 보입니다. 이것이 파일의 **권한**으로, 소유자·그룹·기타 사용자가 각각 읽기(`r`)·쓰기(`w`)·실행(`x`)을 할 수 있는지를 나타냅니다. 각 권한은 숫자로도 표현하는데, 읽기 4·쓰기 2·실행 1을 더한 값입니다. 그래서 `rwx`는 7, `r-x`는 5가 됩니다. 권한을 바꾸는 명령은 `chmod`이며, 특히 스크립트를 실행하려면 실행 권한을 주는 `chmod +x`가 꼭 필요합니다.

```bash
chmod +x script.sh    # 실행 권한 부여
chmod 644 data.csv    # 소유자 rw, 나머지 r
```

### 쉘 스크립트 — 반복을 자동화하기

매번 손으로 치는 명령을 파일에 모아 두면, 한 번의 실행으로 자동화할 수 있습니다. 이것이 **쉘 스크립트**입니다. 첫 줄의 `#!/bin/bash`는 "이 파일을 bash로 실행하라"는 표시(셔뱅)이고, 그 아래로 변수·조건문·반복문을 평소처럼 씁니다.

```bash
#!/bin/bash
for f in *.csv; do          # 모든 csv 파일에 대해
  echo "처리 중: $f"
  wc -l "$f"                # 줄 수를 센다
done
```

이렇게 만든 스크립트는 `chmod +x`로 실행 권한을 준 뒤 `./스크립트.sh`로 돌립니다. 반복되는 데이터 작업을 스크립트로 묶어 두는 습관이, 나중에 본격적인 파이프라인 자동화로 이어집니다.
