# 정답 해설 — analyze_log.sh

## 핵심 아이디어
모든 항목이 **"필드 추출 → 정렬 → 집계"** 패턴입니다.

| 항목 | 명령 | 설명 |
|------|------|------|
| 총 요청 수 | `wc -l < "$LOG"` | 줄 수 = 요청 수. `<` 로 넘기면 파일명이 안 붙어 숫자만 깔끔하게 출력 |
| 상태코드 빈도 | `awk '{print $9}' \| sort \| uniq -c \| sort -rn` | 9번째 필드 추출 후 정렬→중복카운트→빈도 역순 |
| 에러 수 | `awk '$9 >= 400' \| wc -l` | awk 조건식으로 400 이상만 통과시킨 뒤 줄 수 |
| IP Top 5 | `awk '{print $1}' \| sort \| uniq -c \| sort -rn \| head -5` | 1번째 필드(IP) 빈도 상위 5 |

## 왜 `sort` 다음에 `uniq` 인가?
`uniq`는 **인접한** 중복만 제거/집계합니다. 먼저 `sort`로 같은 값을 모아두지 않으면
빈도가 잘못 계산됩니다. 그래서 항상 `sort | uniq -c` 순서입니다.
