## 📦 프로젝트 개요

### 주제
**"동네 가게 판매 데이터 분석"** — 가상의 매장 판매 기록(`data/sales_raw.csv`)이 주어집니다.
이 데이터는 현실처럼 지저분합니다: 결측치, 들쭉날쭉한 대소문자/공백, 잘못된 숫자 형식,
중복 행이 섞여 있습니다.

### 분석 질문 (예시 — 자유롭게 확장 가능)
1. 어떤 상품 카테고리가 매출이 가장 높은가?
2. 요일/날짜별 매출 추이는 어떤가?
3. 결제수단(`payment`)별 비중은?
4. 데이터 품질 문제는 무엇이었고 어떻게 처리했는가?

### 파이프라인 단계 (collect → clean → analyze → report)
```
data/sales_raw.csv
   │  (1) load      : CSV 읽기
   │  (2) clean     : 결측/중복/형식 정제, 타입 변환
   │  (3) analyze   : groupby 집계, 분석 질문에 답
   │  (4) visualize : 막대/선 그래프 저장
   ▼
outputs/  (summary.json, *.png) + REPORT.md
```

## 🛠 요구사항 (산출물)
`exercises/`의 골격을 채워 아래를 완성합니다.

1. **`pipeline.py`** — load/clean/analyze 함수를 구현 (`check.py`로 핵심 로직 검증)
2. **정제 리포트** — 어떤 더러운 데이터를 어떻게 고쳤는지 기록
3. **차트 1개 이상** — 카테고리별 또는 날짜별 매출 (matplotlib)
4. **`REPORT.md`** — 분석 질문 3개에 대한 답 + 발견 사실 + 데이터 품질 메모
5. **발표** — 5분 내외로 "무엇을 했고, 무엇을 알아냈는지" 공유
