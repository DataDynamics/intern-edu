# 모듈 14 — 평가 루브릭 (미니 프로젝트)

각 항목 0~3점 (0 미흡 / 1 기본 / 2 양호 / 3 우수). 총 18점.

| 영역 | 평가 내용 | 배점 |
|------|-----------|------|
| 데이터 정제 | 품질 문제를 찾아내고 합리적으로 처리했는가 (`check.py` 통과 포함) | 3 |
| 분석 | 분석 질문에 데이터로 정확히 답했는가 | 3 |
| 시각화 | 적절한 차트로 핵심을 전달했는가 | 3 |
| 리포트 | REPORT.md가 "무엇을/왜/그래서"를 담았는가 | 3 |
| 코드 품질 | 함수 분리·가독성·재현성(다시 실행해도 같은 결과) | 3 |
| Git/발표 | 브랜치+PR, 깔끔한 커밋, 발표 전달력 | 3 |

## 통과 기준
- **합격**: 12점 이상 + `check.py` 통과
- **우수**: 15점 이상 + 분석 질문을 스스로 확장

## 평가 시 보는 신호
- 정제 규칙의 **판단 근거**가 리포트에 적혀 있는가 (단순 코드가 아니라 사고)
- AI 도구를 썼다면, 결과를 **검증**한 흔적이 있는가 (모듈 00·01 연결)
