# 📖 용어집 (Glossary)

> 이 과정 전체에서 반복적으로 등장하는 핵심 용어를 한곳에 모았습니다.
> 모듈을 읽다가 낯선 단어를 만나면 먼저 여기서 찾아보세요. 용어는 주제별로 묶여 있고, 각 주제 안에서는 대체로 모듈에서 처음 나오는 순서대로 정렬했습니다.

> **이 문서의 위치**: 대학생 트랙과 졸업생 트랙이 함께 참조하는 공용 용어집입니다.
> 졸업생 트랙 모듈은 아직 보완 중이며, 본문이 채워지는 대로 이 용어집에 실무 용어가 계속 추가됩니다.

## 진로 · 마인드셋

| 용어 | 설명 |
|------|------|
| **데이터 엔지니어 (Data Engineer)** | 데이터가 흐르는 길(수집 → 저장 → 가공 → 제공)을 설계·구현하고, 그 길이 믿을 수 있게 굴러가도록 운영하는 사람. |
| **데이터 파이프라인 (Data Pipeline)** | 원천에서 데이터를 가져와 정제·가공하여 사용처(분석/서비스)로 흘려보내는 일련의 자동화된 처리 흐름. |
| **AI 레버리지 (AI Leverage)** | AI를 단순 대체재가 아니라 생산성을 끌어올리는 지렛대로 쓰는 것. 코드는 AI가 거들고, 판단·검증·설계는 사람이 맡는 방식. |
| **포트폴리오 (Portfolio)** | 내가 만든 결과물(프로젝트·코드·문서)을 모아 실력을 증명하는 모음. 취업에서 "무엇을 할 수 있는가"를 보여 주는 핵심 자료. |
| **개인 성장 계획서** | 현재 위치와 목표를 점검하고, 무엇을 언제까지 익힐지 스스로 설계하는 학습 로드맵. |

## 개발 환경 · 터미널

| 용어 | 설명 |
|------|------|
| **터미널 / 쉘 (Terminal / Shell)** | 텍스트 명령으로 컴퓨터에 일을 시키는 창(터미널)과 그 명령을 해석해 실행하는 프로그램(쉘, 예: bash·zsh). |
| **명령어 (Command)** | 쉘에게 내리는 한 줄의 지시. 예: `ls`(목록 보기), `cd`(이동), `cat`(내용 보기). |
| **경로 (Path)** | 파일이나 폴더의 주소. **절대경로**는 최상위(`/`)부터 적은 전체 주소, **상대경로**는 현재 위치를 기준으로 적은 주소. |
| **작업 디렉토리 (Working Directory)** | 현재 쉘이 "서 있는" 폴더. 상대경로는 모두 이 위치를 기준으로 해석된다. |
| **VSCode** | 가장 널리 쓰이는 코드 에디터. 코드 작성·터미널·확장 기능을 한 화면에서 다룬다. |
| **가상환경 (Virtual Environment)** | 프로젝트마다 독립된 Python·패키지 공간을 따로 두는 것. 프로젝트 간 버전 충돌을 막는다(`venv`, `uv`). |
| **uv / venv** | Python 가상환경과 패키지를 만들고 관리하는 도구. `uv`는 빠른 최신 도구, `venv`는 Python 기본 내장 도구. |
| **패키지 (Package)** | 남이 만들어 둔 재사용 가능한 코드 묶음. `pip`/`uv`로 설치해 가져다 쓴다(예: `pandas`). |
| **WSL2 + Ubuntu** | 윈도우 안에서 리눅스(Ubuntu)를 돌리는 환경. 윈도우 사용자가 리눅스 실습을 그대로 따라 할 수 있게 해 준다. |

## Markdown · 문서

| 용어 | 설명 |
|------|------|
| **Markdown (마크다운)** | 평범한 텍스트에 `#`, `*`, `-` 같은 기호만 더해 제목·목록·강조를 표현하는 가벼운 서식 언어. README가 대표적. |
| **헤더 (Header)** | `#` 개수로 단계를 나타내는 제목. `#` 하나는 문서 대표 제목(한 문서에 하나), `##`·`###`로 하위 단계를 만든다. |
| **인라인 코드 / 코드 블록** | 문장 속 짧은 코드는 백틱(`` ` ``)으로, 여러 줄 코드는 백틱 세 개(```` ``` ````)로 감싸 보여 준다. |
| **링크 / 이미지** | 링크는 `[보일 글자](주소)`, 이미지는 그 앞에 `!`를 붙인 `![설명](주소)` 형식. |

## Linux

| 용어 | 설명 |
|------|------|
| **쉘 스크립트 (Shell Script)** | 여러 명령어를 파일 하나에 적어 두고 한 번에 실행하는 작은 프로그램(`.sh`). 반복 작업 자동화의 출발점. |
| **파이프 (Pipe)** | 한 명령의 출력을 다음 명령의 입력으로 넘기는 연결. |
| **리다이렉션 (Redirection)** | 명령의 출력을 화면 대신 파일로 보내거나(`>`, `>>`), 파일을 입력으로 받는 것(`<`). |
| **권한 (Permission)** | 파일·디렉토리에 대한 읽기(r)·쓰기(w)·실행(x) 허용 여부. `chmod`로 바꾼다. |
| **표준 입출력 (stdin / stdout / stderr)** | 프로그램의 기본 입력(stdin), 정상 출력(stdout), 오류 출력(stderr) 통로. 파이프·리다이렉션이 이 통로를 연결한다. |

## Git · 협업

| 용어 | 설명 |
|------|------|
| **Git** | 코드의 변경 역사를 기록·관리하는 버전 관리 도구. "언제·누가·무엇을 바꿨는가"를 추적한다. |
| **저장소 (Repository, repo)** | Git이 관리하는 프로젝트 단위. 내 컴퓨터의 **로컬 저장소**와 GitHub 등에 있는 **원격 저장소**가 있다. |
| **커밋 (Commit)** | 변경 사항을 하나의 의미 있는 묶음으로 저장소에 기록하는 것. 되돌아갈 수 있는 "저장 지점". |
| **스테이징 영역 (Staging Area)** | 커밋에 포함할 변경만 골라 담아 두는 대기 공간. `git add`로 올린다. (작업 디렉토리 → 스테이징 → 저장소) |
| **브랜치 (Branch)** | 본류에 영향을 주지 않고 따로 작업하기 위해 갈라낸 가지. 기능별로 나눠 작업한 뒤 다시 합친다. |
| **머지 (Merge)** | 갈라진 브랜치의 변경을 하나로 합치는 것. |
| **충돌 (Conflict)** | 같은 부분을 서로 다르게 바꿔 자동으로 합칠 수 없을 때 발생. 사람이 직접 골라 해결해야 한다. |
| **Pull Request (PR)** | 내 브랜치의 변경을 본류에 합쳐 달라고 요청하며 리뷰를 받는 협업 절차. |

## Python

| 용어 | 설명 |
|------|------|
| **변수 (Variable)** | 값에 붙인 이름표. `x = 10`처럼 값을 담아 두고 이름으로 꺼내 쓴다. |
| **자료형 (Type)** | 값의 종류. 정수(int)·실수(float)·문자열(str)·불리언(bool) 등. |
| **연산자 (Operator)** | 값을 계산·비교하는 기호. 산술(`+ - * /`), 비교(`== < >`), 논리(`and or not`). |
| **조건문 (if)** | 조건이 참인지에 따라 실행 흐름을 가르는 구문(`if`/`elif`/`else`). |
| **반복문 (Loop)** | 같은 일을 여러 번 되풀이하는 구문(`for`, `while`). |
| **자료구조 (Data Structure)** | 값을 모아 두는 그릇. 리스트(list)·튜플(tuple)·딕셔너리(dict)·집합(set). |
| **함수 (Function)** | 입력을 받아 일을 하고 결과를 돌려주는, 이름 붙은 코드 묶음. "하나의 함수는 하나의 일"이 원칙. |
| **모듈 (Module)** | 함수·변수를 모아 둔 `.py` 파일. `import`로 불러와 재사용한다. |
| **예외 처리 (Exception Handling)** | 실행 중 발생하는 오류를 잡아 프로그램이 멈추지 않게 다루는 것(`try`/`except`). |

## 파일 · 데이터 포맷

| 용어 | 설명 |
|------|------|
| **인코딩 (Encoding)** | 글자를 바이트로 바꾸는 규칙(예: UTF-8). 한글 깨짐을 막으려면 읽고 쓸 때 항상 명시한다. |
| **CSV** | 쉼표로 값을 구분해 표 형태 데이터를 담는 텍스트 파일. 엑셀·데이터 교환의 사실상 표준. |
| **JSON** | 키-값과 중첩 구조로 데이터를 표현하는 텍스트 포맷. 웹 API와 설정 파일에서 널리 쓰인다. |
| **파싱 (Parsing)** | 텍스트 형태의 데이터를 프로그램이 다루기 좋은 구조(리스트·딕셔너리 등)로 해석해 변환하는 것. |

## SQL · 데이터베이스

| 용어 | 설명 |
|------|------|
| **데이터베이스 (Database)** | 데이터를 체계적으로 저장·조회·관리하는 시스템. |
| **관계형 데이터베이스 (RDB)** | 데이터를 여러 테이블에 나눠 담고 키로 연결하는 데이터베이스(예: SQLite, PostgreSQL). |
| **SQL (Structured Query Language)** | 데이터베이스에 질문을 던져 원하는 데이터를 꺼내고 다루는 표준 언어. |
| **테이블 (Table)** | 행(row)과 열(column)로 이루어진 표. 보통 하나의 대상(고객, 주문 등)을 담는다. |
| **기본키 (Primary Key, PK)** | 테이블의 각 행을 유일하게 식별하는 열. 보통 `id`. |
| **외래키 (Foreign Key, FK)** | 다른 테이블의 기본키를 가리켜 테이블끼리 관계를 맺는 열. |
| **SELECT / WHERE / ORDER BY** | 조회의 기본. `SELECT`로 열을 고르고, `WHERE`로 조건에 맞는 행만 추리며, `ORDER BY`로 정렬한다. |
| **JOIN** | 외래키를 기준으로 흩어진 두 테이블을 이어 붙여 함께 조회하는 것. `INNER`/`LEFT` 등 종류가 있다. |
| **GROUP BY / 집계 함수** | 같은 값끼리 묶어(`GROUP BY`) 무리별 통계를 내는 것. `COUNT`·`SUM`·`AVG`·`MIN`·`MAX`. |
| **NULL** | "값이 비어 있음". 0이나 빈 문자열과 다르며 `IS NULL`/`IS NOT NULL`로 다룬다. |
| **서브쿼리 (Subquery)** | 쿼리 안에 들어 있는 또 다른 쿼리. 안쪽 결과를 바깥 쿼리가 받아 쓴다. |
| **윈도우 함수 (Window Function)** | 행을 묶어 하나로 줄이지 않고, 각 행 옆에 그룹 단위 계산값을 나란히 붙이는 함수. |
| **인덱스 (Index, 색인)** | 자주 조회하는 열에 미리 만들어 두는 "찾아보기". 검색 속도를 끌어올린다. |
| **SQL 방언 (Flavor)** | 표준 SQL을 바탕으로 데이터베이스마다 조금씩 다른 문법 차이. |

## Docker

| 용어 | 설명 |
|------|------|
| **Docker** | 애플리케이션과 실행 환경을 통째로 묶어 어디서나 똑같이 실행하게 해 주는 컨테이너 도구. |
| **이미지 (Image)** | 컨테이너를 찍어 내는 틀. 실행에 필요한 코드·라이브러리·설정이 담긴 읽기 전용 묶음. |
| **컨테이너 (Container)** | 이미지를 실행해 격리된 채 돌아가는 하나의 인스턴스. 가볍고 빠르게 띄우고 지운다. |
| **볼륨 (Volume)** | 컨테이너가 사라져도 데이터를 잃지 않도록 바깥에 따로 저장하는 공간. |
| **docker-compose** | 여러 컨테이너(예: 앱 + 데이터베이스)를 한 파일(`docker-compose.yml`)로 정의해 한 번에 띄우는 도구. |
| **포트 (Port)** | 컨테이너 안 서비스에 바깥에서 접속하기 위해 연결하는 통로 번호(예: `5432`). |

## pandas · 데이터 분석

| 용어 | 설명 |
|------|------|
| **pandas** | 표 형태 데이터를 다루는 Python 라이브러리. 데이터 정제·가공·분석의 핵심 도구. |
| **DataFrame** | pandas의 표 자료구조. 행과 열로 이루어진, 엑셀 시트 같은 2차원 데이터. |
| **Series** | DataFrame의 한 열에 해당하는 1차원 데이터. |
| **불리언 인덱싱 (Boolean Indexing)** | 참/거짓 조건으로 원하는 행만 골라내는 방법. 예: `df[df.amount > 10000]`. |
| **결측치 (Missing Value)** | 비어 있는 값(`NaN`). 채우거나 제거하는 등 정제 과정에서 다룬다. |
| **EDA (Exploratory Data Analysis, 탐색적 데이터 분석)** | 데이터를 본격 분석하기 전에 분포·관계·이상치를 살피며 "어떤 데이터인지" 감을 잡는 단계. |
| **히스토그램 / 산점도 (Histogram / Scatter)** | 분포를 보는 히스토그램, 두 변수의 관계를 보는 산점도 등 대표적인 시각화 차트. |
| **데이터 정제 (Data Cleaning)** | 더러운 원본 데이터에서 오류·중복·결측치를 바로잡아 분석 가능한 상태로 다듬는 작업. |

## 데이터 엔지니어링 (졸업생 트랙)

> 졸업생 트랙 본문이 보완되면 이 영역에 용어가 계속 추가됩니다.

| 용어 | 설명 |
|------|------|
| **모던 데이터 스택 (Modern Data Stack)** | 수집·적재·변환·오케스트레이션·웨어하우스로 이어지는 오늘날의 표준 데이터 도구 묶음. |
| **ETL / ELT** | 데이터를 옮기는 세 단계의 순서. ETL은 추출(Extract)→변환(Transform)→적재(Load), ELT는 추출→적재→변환(웨어하우스 안에서 변환). |
| **데이터 모델링 (Data Modeling)** | 데이터를 어떤 테이블·관계로 구조화할지 설계하는 일. |
| **오케스트레이션 (Orchestration)** | 여러 작업(태스크)의 실행 순서·일정·의존 관계를 자동으로 조율하는 것(예: Airflow). |
| **워크플로우 (Workflow)** | 정해진 순서와 의존 관계에 따라 묶여 실행되는 작업들의 흐름. |
| **배치 vs 스트리밍 (Batch vs Streaming)** | 데이터를 일정 묶음 단위로 모아 처리하는 방식(배치)과, 들어오는 즉시 흘려보내며 처리하는 방식(스트리밍). |
| **데이터 품질 (Data Quality)** | 데이터가 정확·완전·일관·최신인지를 가리키는 신뢰도. 검증 규칙으로 지킨다. |
| **관측성 (Observability)** | 파이프라인의 상태·지표·로그를 들여다보며 문제를 빠르게 파악할 수 있는 정도. |
| **데이터 웨어하우스 (Data Warehouse, DW)** | 분석을 위해 정제된 데이터를 한곳에 모아 두는 대규모 저장소. |
| **분석 엔지니어링 (Analytics Engineering)** | 웨어하우스 안의 원천 데이터를 분석하기 좋은 모델로 변환·관리하는 영역(예: dbt). |
