# 콘텐츠 작성 가이드 (강사용)

이 저장소에 강의·실습 콘텐츠를 추가/수정하는 규칙입니다.

## 디렉토리 & 네이밍 규칙
- 모듈 폴더는 **숫자 prefix + kebab-case 영문**: `04-linux-basics` (정렬·순서가 한눈에)
- 파일/폴더명은 **영문 kebab-case** (경로 깨짐 방지), 문서 **내용은 한국어** OK
- 트랙 간 중복이 쌓이면 `common/modules/`로 추출하는 것을 검토

## 모듈 표준 구조
모든 실습 모듈은 동일한 구조를 따릅니다.

```
modules/NN-주제/
├── README.md          # 학습목표 · 개념설명(강의문서) · 완료 기준
├── examples/          # 강사가 보여주는 실행 가능한 예제 코드
├── exercises/
│   ├── starter/       # 인턴이 채우는 빈 골격 (TODO 포함)
│   └── solution/      # 정답
├── assessment/        # 퀴즈 또는 완료 기준 체크리스트
└── data/              # (선택) 이 모듈 전용 데이터
```

작성·토론 중심의 **문서형 모듈**(예: AI 시대 모듈)은 `exercises/` 대신 `worksheet/`를 둡니다.

```
modules/NN-주제/
├── README.md
└── worksheet/         # 자기진단·계획서 등 작성 양식
```

## 모듈 README 템플릿
각 모듈 `README.md`는 다음 섹션을 채웁니다.

```markdown
# 모듈 NN — 제목

> 포커스: 한 줄 요약
> 예상 기간: 1주

## 🎯 학습 목표
## 📚 핵심 주제
## 🛠 실습 / 산출물
## ✅ 완료 기준 (체크리스트)
## 📂 폴더 구성
## 🔗 참고 자료
```

## 실습 코드 규칙
- `exercises/starter/`에는 동작하는 골격 + `# TODO:` 주석으로 채울 부분을 명확히 표시
- `exercises/solution/`은 starter와 **동일한 파일 구조**로, 정답만 채워 넣음
- 가능하면 `examples/`와 `solution/`은 실제로 실행되는 코드로 유지

## 데이터셋
- 공유 데이터는 `datasets/`에, 모듈 전용 데이터는 모듈 내 `data/`에 둡니다.
- 모든 데이터는 **출처와 라이선스**를 README에 명시합니다.

## 기여 흐름
1. 브랜치 생성 → 모듈 작성/수정
2. starter/solution이 의도대로 동작하는지 확인
3. PR 생성 → 리뷰 → 머지
