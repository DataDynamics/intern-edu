# 커리큘럼 전체 개요

데이터 엔지니어 인턴 교육 프로그램의 전체 커리큘럼입니다. 두 트랙 모두
**"왜 어려운가(현실 진단) → 어떻게 살아남는가(전략) → 그래서 데이터 엔지니어링(실행)"**
흐름으로 설계되었습니다.

설계 원칙:
1. **트랙 분리 + 공통 자원 공유** — 콘텐츠는 트랙별로, 환경·데이터셋·참고자료는 공유
2. **모듈 단위 표준화** — 모든 모듈이 동일한 내부 구조
3. **starter / solution 분리** — 자가학습과 채점 모두 지원

### 모듈 00 — AI 시대, 왜 취업이 어려운가 (현실 진단)
> 막연한 불안을 구체적 현실 인식으로 바꿔 학습 동기를 만든다. 겁주기가 아니라 "그래서 무엇을 해야 하는가"로 연결.
- 생성형 AI가 바꾼 신입 개발자 시장: 단순 코딩의 희소가치 하락
- 주니어 채용 감소의 구조 ("경력 같은 신입" 요구)
- 양극화: AI를 도구로 쓰는 사람 vs 대체되는 사람
- "코딩 자체"의 가치 하락 ≠ "문제 해결 능력"의 가치 하락
- 데이터 직군이 상대적으로 견고한 이유 → 이 교육의 방향성
- **활동**: 자기 진단 체크리스트 + 조별 토론

### 모듈 01 — AI 시대, 개발자로 살아남는 법 (생존 전략)
> 위기의식을 실행 가능한 전략으로 전환. 이후 학습을 "왜 하는지" 알고 임하게 만든다.
- AI를 대체재가 아닌 증폭기로 (검증·판단은 사람의 몫)
- 대체되지 않는 역량: 문제 정의력, 데이터·도메인 이해, 시스템 설계·디버깅, 협업
- T자형 인재: 넓은 기초 + 데이터 엔지니어링이라는 깊은 한 축
- AI 시대의 학습법: "왜"를 묻기, AI 답변을 비판적으로 검증, 빠른 프로토타이핑
- 포트폴리오·기록의 힘 (GitHub로 사고 과정 증명)
- **활동**: 개인 성장 계획서 (12주 후 목표 + AI 활용 원칙 + 포트폴리오 계획)

### 모듈 00 — AI 시대 데이터 엔지니어 생존 전략
> 대학생 트랙이 마인드셋·진로라면, 졸업생 트랙은 실무 생산성·커리어 전략에 초점.
- AI를 실무 레버리지로 (ETL 보일러플레이트·SQL·테스트 생성, 사람은 설계·검증에 집중)
- DE가 AI 시대에 강한 이유 (회사 고유 데이터, 신뢰성·품질·도메인 맥락)
- 대체되지 않는 실무 역량: 시스템 설계, 장애 대응, 데이터 신뢰성, 비용·성능 판단
- AI 시대의 안티패턴: 무검증 머지, 맥락 없는 복붙, 환각 로직 → 검증 습관
- 커리어: 구현자에서 데이터 플랫폼 설계자/문제 해결자로 포지셔닝
- **활동**: "우리 팀의 AI 활용 원칙" 초안 작성

## 두 트랙의 연결
- 대학생 트랙 수료자가 졸업생 트랙으로 자연스럽게 이어지도록, 졸업생 00\~01은 "빠른 복습"으로 둠
- 겹치는 깊은 기초는 대학생 트랙 모듈을 참조
- `datasets/`의 동일 샘플 데이터를 대학생은 **분석**, 졸업생은 **파이프라인 대상**으로 활용
