# 인턴 채용 면접 가이드 (면접관용)

Data Dynamics 데이터 엔지니어링 인턴 채용을 위한 면접 질문서와 진행 가이드입니다.

> ⚠️ **취급 주의**: 채용 평가용 자료입니다. 지원자에게 사전 공유하지 마세요.

| 문서 | 용도 |
|------|------|

| [behavioral-questions.md](behavioral-questions.md) | 인성·컬처핏·성장 가능성 질문 |

## 🎯 인턴 채용에서 보는 것

신입/인턴은 **현재 실력보다 성장 가능성과 학습 태도**가 더 중요합니다.
정답 여부보다 **사고 과정, 모르는 것을 다루는 방식, 커뮤니케이션**을 중점으로 평가하세요.

- ✅ 기초 개념을 자기 언어로 설명할 수 있는가
- ✅ 모르는 문제 앞에서 어떻게 접근·질문하는가
- ✅ 피드백을 받아들이고 스스로 학습한 경험이 있는가
- ✅ 팀으로 일할 수 있는 커뮤니케이션·협업 태도가 있는가

## ⏱️ 진행 흐름 (약 45~60분 기준)

| 단계 | 시간 | 내용 |
|------|------|------|
| 아이스브레이킹 | 5분 | 자기소개, 긴장 완화 |
| 인성·동기 | 10~15분 | [behavioral-questions.md](behavioral-questions.md) 중 2~3개 |
| 직무 기술 | 20~25분 | [data-engineering-questions.md](data-engineering-questions.md), 난이도 점진적 상향 |
| 지원자 질문 | 5~10분 | 역질문 — 회사·직무에 대한 관심도 확인 |

## 📝 평가 기준 (1~5점)

| 항목 | 1점 | 3점 | 5점 |
|------|-----|-----|-----|
| 기초 지식 | 개념 부정확 | 핵심 개념 설명 가능 | 원리까지 이해 |
| 문제 해결 | 접근 못 함 | 힌트로 접근 | 스스로 구조화 |
| 학습 태도 | 수동적 | 학습 경험 있음 | 능동적·지속적 학습 |
| 커뮤니케이션 | 의사 전달 어려움 | 명확히 전달 | 듣는 사람 고려 |
| 컬처핏 | 가치관 불일치 | 무난 | 팀에 긍정적 기여 기대 |

## ⚖️ 면접 시 유의사항

- 합법·공정 채용을 위해 **출신지·성별·나이·결혼·가족·종교** 등 직무 무관 질문은 금지합니다.
- 압박보다 **지원자가 사고 과정을 보여줄 수 있도록** 유도하세요.
- 모르는 답에는 "괜찮습니다, 어떻게 접근해볼지 말씀해 주세요"로 사고 과정을 끌어내세요.
