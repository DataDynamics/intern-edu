> 각 질문에는 **확인 포인트(✅)** 와 **꼬리 질문(↳)** 을 함께 적었습니다.

## 1. Python 기초

**Q1. `list`와 `tuple`의 차이를 설명해 주세요.**
- ✅ 가변/불변(mutable/immutable) 개념 이해
- ↳ 언제 tuple을 쓰는 게 좋을까요?

**Q2. 다음 코드의 출력은? 그렇게 되는 이유는?**
```python
def add(x, items=[]):
    items.append(x)
    return items

print(add(1))
print(add(2))
```
- ✅ 기본 인자 mutable 함정(`[1]`, `[1, 2]`) 이해
- ↳ 의도대로 동작하게 하려면 어떻게 고칠까요?

**Q3. 리스트에서 중복을 제거하려면 어떻게 하겠어요?**
- ✅ `set` 활용, 순서 보존 시 trade-off 인지
- ↳ 원래 순서를 유지해야 한다면?

## 2. SQL

**Q4. `INNER JOIN`과 `LEFT JOIN`의 차이를 예로 설명해 주세요.**
- ✅ 매칭되지 않는 행 처리 차이 이해
- ↳ 주문이 없는 고객까지 모두 보려면 어떤 JOIN을 쓸까요?

**Q5. `WHERE`와 `HAVING`의 차이는?**
- ✅ 집계 전/후 필터링, `GROUP BY`와의 관계
- ↳ "주문이 3건 이상인 고객"은 어디서 거를까요?

**Q6. 각 부서별 평균 급여를 구하는 쿼리를 작성해 주세요. (테이블: `employees(id, name, dept, salary)`)**
- ✅ `GROUP BY`, 집계 함수 사용
```sql
SELECT dept, AVG(salary) AS avg_salary
FROM employees
GROUP BY dept;
```
- ↳ 평균 급여가 5000 이상인 부서만 보려면?

**Q7. 쿼리가 느릴 때 가장 먼저 무엇을 확인하겠어요?**
- ✅ 인덱스, 실행 계획(`EXPLAIN`), 풀스캔 여부 등 언급
- ↳ 인덱스를 무조건 많이 거는 게 좋을까요?

## 3. 데이터 파이프라인 / ETL

**Q8. ETL과 ELT의 차이를 설명해 주세요.**
- ✅ 변환 시점(적재 전/후), 각 방식의 장단점
- ↳ 클라우드 데이터 웨어하우스 시대에 ELT가 많아진 이유는?

**Q9. 매일 외부 API에서 데이터를 받아 DB에 저장하는 파이프라인을 만든다면, 어떤 단계로 설계할까요?**
- ✅ 수집 → 검증 → 변환 → 적재, 실패/재시도 고려
- ↳ API가 가끔 실패한다면 어떻게 대응할까요?
- ↳ 어제 데이터를 다시 받아야 한다면(멱등성)?

**Q10. 배치(batch)와 스트리밍(streaming) 처리의 차이와 각각의 적합한 사례는?**
- ✅ 지연/처리량 trade-off, 사용 사례 예시
- ↳ 실시간 이상 거래 탐지는 어느 쪽이 맞을까요?

## 4. 데이터 모델링 / 품질

**Q11. 정규화(normalization)는 왜 하고, 어떤 단점이 있나요?**

- ↳ 분석용 테이블에서 일부러 비정규화하는 이유는?

**Q12. 데이터 품질을 보장하려면 어떤 점들을 검증하겠어요?**
- ✅ NULL/중복/범위/스키마/신선도(freshness) 등
- ↳ 파이프라인에서 자동으로 검증하려면 어디에 넣을까요?
